4. 计算输出 shape，并写回 `m/n/k` 辅助属性。

## 2.2 `inferShape` 完整实现

实现步骤如下：

1. 输入校验：必须 2 个输入，且二者 rank 均 >= 2。
2. 按 `transA/transB` 计算矩阵维：
   - `mA`、`kA` 来自 A 最后两维（是否交换取决于 `transA`）
   - `kB`、`nB` 来自 B 最后两维（是否交换取决于 `transB`）
3. 断言 `kA == kB`，保证可乘。
4. 提取 batch 维：
   - `batchA = A[:-2]`
   - `batchB = B[:-2]`
5. `outBatch = infer_broadcast(batchA, batchB)`
6. 输出 shape = `outBatch + [mA, nB]`
7. 写回内部辅助字段：
   - `m = mA`
   - `n = nB`
   - `k = kA`
8. 返回 `{{outShape}}`

## 3. 形状推导规则说明

## 3.1 矩阵部分

对最后两维，按是否转置后解释：

1. A: `(m, k)`
2. B: `(k, n)`
3. 输出: `(m, n)`

其中：

- `transA=false`: A 最后两维按 `[..., m, k]`
- `transA=true`:  A 最后两维按 `[..., k, m]`
- `transB=false`: B 最后两维按 `[..., k, n]`
- `transB=true`:  B 最后两维按 `[..., n, k]`

## 3.2 Batch 维部分

A 与 B 的前 `rank-2` 维按 ONNX 多向广播规则对齐，得到输出 batch 维。

最终输出：

- `broadcast(batchA, batchB) + [m, n]`

## 4. 测试用例与实现对应关系

`test_matmul` 里的典型场景：

1. 同 batch、不转置：
   - A `{1,3,5}`, B `{1,5,2}` -> 输出 `{1,3,2}`
2. 单边转置：
   - A `{3,5,4}`, `transA=true` -> 视作 `{3,4,5}`
   - B `{3,5,2}` -> 输出 `{3,4,2}`
3. batch 广播：
   - A `{1,2,3,5}`, B `{1,1,5,2}` -> `{1,2,3,2}`
4. 更复杂 batch + transpose 组合：
   - 覆盖 `transA/transB` 同时启用

本次实现与上述场景全部一致。

## 6. 复杂度分析

设：

- `RA` = rank(A)
- `RB` = rank(B)
- `R = max(RA-2, RB-2)`（batch 广播相关）

则：

1. 时间复杂度：`O(R)`（主要在 batch 广播）
2. 空间复杂度：`O(R)`（输出 batch shape）

## 7. 健壮性与错误处理

本次实现提供了关键约束断言：

1. 输入数量必须是 2。
2. rank 必须 >= 2。
3. 内积维必须匹配（`kA == kB`）。

这能保证非法 Matmul 在构图阶段尽早失败。
