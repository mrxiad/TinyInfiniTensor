## 2. 代码改动

文件：`src/operators/concat.cc`

函数：`ConcatObj::inferShape(const TensorVec &inputs)`

新增/实现内容：

1. `IT_ASSERT(!inputs.empty())`：防止空输入。
2. 校验 `dim` 在 `[0, rank)` 范围。
3. 以第一个输入 shape 作为基准 `dims`。
4. 遍历其余输入：
   - 校验 rank 一致。
   - 对非拼接轴，强制与基准轴长度一致。
   - 对拼接轴，执行 `dims[axis] += cur[axis]`。
5. 返回 `{{dims}}`。

## 3. 算法说明

## 3.1 Concat 的 shape 规则

设输入为 `N` 个张量，拼接轴为 `dim`：

1. 所有输入 rank 必须相同。
2. 对任意 `axis != dim`，各输入维度必须一致。
3. 输出在 `dim` 上的长度是所有输入该轴长度之和。

公式化表示：

- `out[axis] = in0[axis]`（`axis != dim`）
- `out[dim] = sum(in_i[dim])`

## 3.2 为什么要做严格断言

如果非拼接轴长度不一致，运行期内核无法定义合法映射，
因此必须在构图阶段就拒绝非法 shape 组合，避免后续隐蔽崩溃。

## 4. 与框架执行链路关系

`ConcatObj` 构造时会调用 `checkValid`：

2. 框架据此创建输出 tensor。
3. nativecpu concat kernel 根据输入输出 shape 做实际拷贝。

## 6. 复杂度分析

设：

- 输入个数 `N`
- rank `R`

则 `inferShape` 复杂度为：

1. 时间复杂度：`O(N * R)`
2. 空间复杂度：`O(R)`（输出 shape）
