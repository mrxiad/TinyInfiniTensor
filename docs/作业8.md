# 作业8总结：图优化规则实现（详细版）

## 1. 作业目标

作业8要求在 `GraphObj::optimize()` 中实现两类图优化规则：

1. 去除冗余算子：
   - 典型例子是两个相邻 transpose 互为逆操作时可消除
2. 合并算子：
   - 典型例子是将 transpose 融入 matmul 的 `transA/transB`

对应文件：

- `src/core/graph.cc`

对应测试：

- `test/core/test_graph.cc`（`test_graph`）

## 2. 实现概览

本次在 `GraphObj::optimize()` 中实现了完整优化流程，核心由 5 部分组成：

1. 规则判定工具函数
2. 节点删除与张量删除工具
3. 图连接关系重建函数
4. 两条优化规则的迭代应用
5. 收尾清理与拓扑重排

## 3. 关键实现细节

## 3.1 规则判定工具

在 `optimize()` 内定义了两个判定函数：

1. `isInversePermute(p1, p2)`
   - 判定 `p2` 是否是 `p1` 的逆置换
   - 用于识别“连续 transpose 抵消”

2. `isLastTwoSwap(perm, rank)`
   - 判定 `perm` 是否“仅交换最后两维，其他维保持不变”
   - 用于识别可融合进 matmul 的 transpose

## 3.2 图修改辅助工具

新增两个删除辅助：

1. `eraseOp(victim)`：从 `ops` 容器移除指定算子
2. `eraseTensor(victim)`：从 `tensors` 容器移除指定张量

以及关键重建函数：

- `rebuildConnections()`

作用：

1. 清空所有 tensor 的 `source/targets`
2. 清空所有 op 的 `predecessors/successors`
3. 重新根据 `op->outputs` 建立 tensor source
4. 重新根据 `op->inputs` 建立 tensor targets 与 op 前驱后继

这一步保证每次图改写后连接状态一致，避免脏边。

## 3.3 规则1：连续逆 transpose 消除

匹配条件：

1. 当前节点是 transpose（记为 `trans2`）
2. `trans2` 的输入来源也是 transpose（记为 `trans1`）
3. `trans1` 的输出只被 `trans2` 消费
4. `trans1.permute` 与 `trans2.permute` 互为逆

改写动作：

1. 将 `trans2` 输出张量的所有下游输入替换为 `trans1` 的输入张量
2. 删除 `trans1` 与 `trans2`
3. 删除中间张量（`trans1` 输出、`trans2` 输出）
4. 重新构建连接

## 3.4 规则2：transpose 融合到 matmul

匹配条件：

1. 节点是 matmul
2. matmul 某个输入来自 transpose
3. 该 transpose 输出只被该 matmul 使用
4. transpose 仅交换最后两维（`isLastTwoSwap`）

改写动作：

1. matmul 输入替换为 transpose 的原输入
2. 若融合的是 A 侧：`transA = !transA`
3. 若融合的是 B 侧：`transB = !transB`
4. 删除 transpose 算子与其输出张量
5. 重新构建连接

## 3.5 迭代应用与收尾

优化采用 `while(changed)` 循环，直到一次遍历没有命中规则：

1. 先尝试规则1
2. 若无变化再尝试规则2
3. 每次变化后重建连接并重新开始

最终：

1. 删除孤立张量（无 source 且无 targets）
2. 再次重建连接
3. `sorted = false` 后执行一次 `topo_sort()`

## 4. 与测试用例的对应

`test_graph` 构图：

1. `i1 -> T1 -> T2 -> ...`
2. `i2 -> T3 -> ...`
3. `Matmul(T2, T3) -> o`

目标期望：

1. `T1` 和 `T2` 抵消并删除
2. `T3` 融合到 Matmul 的 `transB`
3. 最终只剩 1 个 Matmul 与 3 个 tensor（`i1/i2/o`）

本次实现达到该结果，`test_graph` 通过。

## 5. 过程中修复的一个实现陷阱

首次实现时出现了“删除连续 transpose 只删掉一个”的问题，原因是：

1. 在遍历 `ops` 的引用变量 `op` 上直接删除容器元素
2. 导致引用失效（迭代器/引用悬空）

修复方式：

- 先保存稳定句柄 `trans2Op = op`
- 再执行 `eraseOp(trans1Op)` 与 `eraseOp(trans2Op)`

修复后 `test_graph` 稳定通过。

## 7. 复杂度分析

设：

- `O` = 算子数
- `T` = 张量数
- `E` = 边数（输入连接）

则：

1. 每轮规则扫描约 `O(O)`
2. `rebuildConnections` 约 `O(T + O + E)`
3. 总复杂度与命中次数相关，近似为 `K * O(T + O + E)`

其中 `K` 为优化迭代轮数（实际一般较小）。

## 8. 健壮性与约束

当前实现包含以下防护：

1. 只有“单消费者”的 transpose 才允许融合/删除，避免破坏多分支依赖。
2. 只融合“最后两维交换”的 transpose，确保与 matmul `transA/transB` 语义严格一致。
3. 每次图改写后强制重建连接，防止 predecessor/successor/source/targets 不一致。
