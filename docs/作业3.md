# 作业3总结：Clip 算子形状推导（详细版）

1. `ClipObj::inferShape` 的实现。
2. 保证 Clip 作为逐元素 unary 算子，不改变张量形状。

对应文件：

- `src/operators/unary.cc`

对应测试：

## 2. 代码改动

文件：`src/operators/unary.cc`

函数：`ClipObj::inferShape(const TensorVec &inputs)`

从“返回 `std::nullopt`”改为：

1. 校验输入个数：`IT_ASSERT(inputs.size() == 1)`
2. 返回与输入相同的 shape：`return {{inputs[0]->getDims()}};`

这是 Clip 在 ONNX 语义中的标准行为：

- Clip 对每个元素做裁剪（min/max）
- 只改变数值，不改变 rank 与各维长度

## 3. 设计理由

## 3.1 为什么只需要返回输入 shape

Clip 是纯逐元素操作：

- 不做 reduction
- 不做维度重排
- 不做拼接/切分

因此输出 shape 必然与输入一致。

## 3.2 为什么加输入个数断言

当前 `ClipObj` 定义是 unary operator：

- `numInputs() == 1`

在 `inferShape` 里显式断言可以更快暴露非法构图输入，避免后续逻辑隐式出错。

## 4. 与框架执行链路的关系

`OperatorObj::checkValid` 在创建算子时会调用 `inferShape`：

1. 如果 `inferShape` 返回 `nullopt`，算子创建失败。
2. 如果返回合法 shape，框架会创建输出 Tensor。

作业3修改之前：

- `ClipObj::inferShape` 返回 `nullopt`
- `g->addOp<ClipObj>(...)` 会失败

作业3修改之后：

- 能正常创建 Clip 输出 tensor
- 输出维度与输入一致
- 测试可以顺利通过
