# 项目现状与能力边界

更新时间：2026-02-22

## 1. 当前状态（实测）

2026-02-22 本地执行 `make test-cpp`：

1. 总测试数：11
2. 通过：11
3. 失败：0

这说明训练营 8 个作业对应的核心路径在当前代码状态下已打通。

## 2. 当前已具备能力

1. 图构建、拓扑排序、shape infer、dtype infer
2. 图级内存分配与复用（`dataMalloc`）
3. 图优化（逆 transpose 消除、transpose 融合 matmul 标志）
4. CPU 运行时 + 部分算子 kernel 执行
   - 已注册示例：Add/Sub/Mul/Div/Relu/Clip/Transpose/Concat

## 3. 当前边界（仍需注意）

1. 设备能力边界
   - 当前主要是 CPU 路径
2. 算子执行边界
   - Matmul 在当前测试中主要覆盖 shape infer 路径，`test_matmul` 不执行 Matmul kernel
   - `src/kernels/cpu` 目录当前无 Matmul kernel 文件
3. ONNX 覆盖边界
   - 只覆盖训练营作业涉及的算子子集，不是完整 ONNX 推理框架

## 4. 文档一致性现状

当前文档存在阶段性记录冲突：

1. `docs/项目分析报告.md` 含“11/11 失败”结论（历史阶段）
2. `docs/作业8.md` 记录“11/11 通过”

以最新本地测试结果为准。建议把 `docs/项目分析报告.md` 视为历史快照而非实时状态页。

## 5. 工程风险（与代码无关但影响协作）

1. CI 触发规则
   - `.github/workflows/build.yml` 中 `pull_request` 仅匹配 `**.md` 与 `LICENSE`
   - 普通代码 PR 可能不自动触发该工作流
2. 工具链文档不完全一致
   - `README.md` 默认 clang-14
   - `docs/项目部署.md` 示例主写 gcc

## 6. 维护建议

1. 每次发布前更新本页中的“测试状态日期和结果”。
2. 关键行为变化后同步更新 `docs/训练营作业介绍.md` 的 TODO 描述状态。
3. 将 `docs/项目分析报告.md` 标记为“历史分析（某日期）”避免误读。
